EasySOA REST Services Documentation

Service notification

Allows to upload any service registry-related information to the Nuxeo repository. Each call allows to register data related to either: an application, a service API, or a service.
Document hierarchy reminder: Services are always contained in Service APIs. Service APIs are either contained in Applications or other APIs.

Protocol POST (notification) or GET (view documentation)
Use .../nuxeo/site/easysoa/notification/(appliimpl or api or service)
Parameters

The request body must contain the parameters in the application/x-www-form-urlencoded format (which is similar to the GET requests parameters). For a list of all available parameters for each document type, view each service's documentation by making a GET request to it.

Example request body: url=http://www.myapp.com/api&parentUrl=http://www.myapp.com/&title=MyApi
Result formatting {"result": ["ok"]}
(if no error was found)
Comments
  • JSONP Support: add a "callback" query parameter to make the request return a JSONP call instead of a JSON object.

WSDL Scraper

Finds WSDLs from a given URL.

Protocol GET
Use .../nuxeo/site/easysoa/wsdlscraper/{url}
Parameters
  • {url} The encoded URL of the page to consider.
Result formatting { "applicationName": "(the application name)", "foundLinks": { "(link name)" : "(link url)", "(link2 name)" : "(link2 url)" ... } }
Comments
  • JSONP Support: add a "callback" query parameter to make the request return a JSONP call instead of a JSON object.
  • Other protocols than HTTP are not supported.
